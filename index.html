Web Storage Guide

Local Storage

Local Storage is a web storage object that allows you to store key-value pairs in the browser with no expiration date.

Property Value
Capacity ~5-10MB (varies by browser)
Persistence Until cleared by user or programmatically
Accessibility Any window/tab from the same origin
Data Type String key-value pairs

Pros:

  • Simple to use
  • Persistent across browser sessions
  • Relatively large storage capacity

Cons:

  • Limited to string data (requires serialization for objects)
  • Synchronous API (can block the main thread)
  • No built-in expiration mechanism

Session Storage

Session Storage is similar to Local Storage but limited to a single browser session. Data is cleared when the session ends (i.e., when the browser tab is closed).

Property Value
Capacity ~5-10MB (varies by browser)
Persistence Until the browser tab is closed
Accessibility Only the window/tab that created it
Data Type String key-value pairs

Pros:

  • Simple to use
  • Isolated to individual tabs
  • Automatically cleared on session end

Cons:

  • Limited to string data
  • Data lost when tab is closed
  • Synchronous API

IndexedDB

IndexedDB is a low-level API for client-side storage of significant amounts of structured data, including files/blobs.

Property Value
Capacity Generally unlimited (browser dependent)
Persistence Until cleared by user or programmatically
Accessibility Any window/tab from the same origin
Data Type JavaScript objects, arrays, files

Pros:

  • Large storage capacity
  • Can store structured data
  • Supports transactions
  • Asynchronous API

Cons:

  • More complex API
  • Steeper learning curve
  • Less browser support for older versions

Cookies

Cookies are small pieces of data stored as text files by the browser. They're commonly used for maintaining user sessions and preferences.

Property Value
Capacity ~4KB per cookie, ~50 cookies per domain
Persistence Can be set to expire or persist indefinitely
Accessibility Any window/tab from the same origin, sent with every HTTP request
Data Type String key-value pairs

Pros:

  • Widely supported
  • Can be accessed server-side
  • Can set expiration dates

Cons:

  • Limited storage capacity
  • Sent with every HTTP request (bandwidth overhead)
  • Can be a security risk if not properly handled

Cache Storage

Cache Storage is part of the Service Worker API and allows for storing of Request/Response object pairs for offline use.

Property Value
Capacity Generally unlimited (browser dependent)
Persistence Until cleared by user or programmatically
Accessibility Through Service Worker
Data Type Request/Response pairs

Pros:

  • Ideal for offline-first applications
  • Can cache entire responses
  • Programmatically controllable

Cons:

  • Requires Service Worker setup
  • More complex to implement
  • Limited to newer browsers

Shared Storage

Shared Storage is a web API that allows limited cross-site storage access while maintaining user privacy. It's designed for specific use cases where information needs to be shared across different sites without compromising privacy or enabling cross-site tracking.

Property Value
Capacity Limited, specific to implementation
Persistence Persistent across sites, subject to browser policies
Accessibility Cross-site, but with restricted access through specific APIs
Data Type Key-value pairs, accessed through specialized worklets

Pros:

  • Enables cross-site data sharing
  • Privacy-preserving by design
  • Suitable for specific use cases like frequency capping
  • Provides alternatives to third-party cookies

Cons:

  • Limited to specific predefined operations
  • No direct read access from main page context
  • Newer API with limited browser support
  • More complex to implement than traditional storage methods
